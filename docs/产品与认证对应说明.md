# 产品与认证对应说明（基于 docs 目录资料）

> 说明：以下内容以 `docs/` 目录中 7 份服务目录文件为基础，整理“每类产品常见的厂家个人认证、渠道/代理认证，以及适用于个人与企业的通用认证”。源文件以产品线、品牌、版本与兼容性为主，未逐条列出“认证名称”，因此认证部分按行业公开信息归纳，供内部规划与对外沟通参考。

## 一、产品分类总览（对应 7 份目录）

## 三、通用认证（个人+企业均常见）

- 公司级资质（对外展示常用）：ISO 9001、ISO 20000、ISO 27001、ITSS 等（来源：官网页面 `leadcom_about_us.html`、`index.html`）。

| 认证类型 | 典型代表 | 说明 |
|---|---|---|
| 信息安全管理体系 | ISO/IEC 27001 | 适用于企业信息安全治理，也可作为人员能力框架参考 |
| IT 服务管理体系 | ISO/IEC 20000 | 面向 IT 服务管理与交付规范化 |
| 质量管理体系 | ISO 9001 | 适用于企业质量管理体系 |
| IT 服务标准 | ITSS（信息技术服务标准） | 国内服务型企业常见能力与成熟度参考 |
| IT 服务管理实践 | ITIL（常见认证如 ITIL 4 Foundation） | 个人与企业均常用，面向流程与服务水平管理 |
| 项目管理 | PMP/PRINCE2 等 | 跨行业通用，项目交付与管理能力证明 |
| 云计算安全 | CCSK/CCSP 等 | 面向云安全治理与架构能力 |
| 数据库通用能力 | 可结合厂商认证与行业认证 | 依据数据库产品与业务场景选择 |

## 四、结合 docs 的“适配/兼容性”提示

- 7 份目录强调的是“产品线 × 操作系统/固件 × 兼容性”，适合做选型与交付规划。
- 源文档中关于“认证”的描述偏向“官方认证兼容/技术认证体系”，因此本文件将其整理为产品→认证路径的通用对照。
- 认证不是这些目录的主线内容，因此“谁家有哪些认证”更适合与厂商最新 Partner Program/Training Portal 对照确认。
- 若用于对外宣传，建议把“认证工程师”与“厂商认证名称”一一对应，并附上官方链接或证书样例。
